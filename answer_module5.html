1. Supervised Learning

Supervised learning is a method where the model is trained using labeled data (input-output pairs). It acts like a student learning under the guidance of a teacher who provides the "answer key."

  • Mechanism: The algorithm identifies patterns between the input features and the known labels. After training, the model is tested with unseen data to predict the output.
  • Mathematical Goal: To learn a mapping function y = f(x) such that for a new x, y can be predicted accurately.
  • Sub-categories:
    • Classification: Predicting discrete categories (e.g., Email: Spam or Not Spam).
    • Regression: Predicting continuous numerical values (e.g., Predicting house prices based on square footage).
  • Example: Handwriting recognition or voice-to-text conversion.

2. Unsupervised Learning

Unsupervised learning occurs when the machine is given unlabeled data and must find hidden structures or patterns within it without any external guidance.

  • Mechanism: Since there is no "teacher" or label, the machine looks for similarities, differences, and patterns. It is used for feature extraction and data compression.
  • Goal: To model the underlying structure or distribution in the data to learn more about it.
  • Sub-categories:
    • Clustering: Grouping similar data points together (e.g., segmenting customers by purchasing habits).
    • Association: Discovering rules that describe your data (e.g., "People who buy bread also tend to buy butter").
  • Example: Recommendation engines (like Netflix suggesting movies based on similar viewing patterns).

3. Reinforcement Learning (RL)

Reinforcement Learning is a feedback-based method where an "Agent" learns to behave in an "Environment" by performing actions and seeing the results.

  • Mechanism: It follows a Trial-and-Error approach. The agent receives a Reward for correct actions and a Penalty (negative reward) for incorrect ones. The agent's objective is to maximize the cumulative reward over time.
  • Key Components: Agent, Environment, State, Action, and Reward.
  • Example: AlphaGo (AI playing board games), self-driving cars navigating traffic, or a robotic arm learning to pick up objects without dropping them.

Decision Tree Classification Algorithm

  • Supervised learning technique can be used for both classification and regression problem.
  • Tree-structured classifier where internal nodes represent the features of a dataset, branches represent the decision rules and each leaf node represents the outcome.
  • To build a tree we use CART algorithm.

Steps to build a tree:

  • Step-1: Begin the tree with the root node, says S, which contains the complete dataset.
  • Step-2: Find the best attribute in the dataset using Attribute Selection Measure (ASM).
  • Step-3: Divide the S into subsets that contains possible values for the best attributes.
  • Step-4: Generate the decision tree node, which contains the best attribute.
  • Step-5: Recursively make new decision trees using the subsets of the dataset created in step -3. Continue this process until a stage is reached where you cannot further classify the nodes and called the final node as a leaf node.

Example:

Suppose there is a candidate who has a job offer and wants to decide whether he should accept the offer or Not. So, to solve this problem, the decision tree starts with the root node (Salary attribute by ASM). The root node splits further into the next decision node (distance from the office) and one leaf node based on the corresponding labels. The next decision node further gets split into one decision node (Cab facility) and one leaf node. Finally, the decision node splits into two leaf nodes (Accepted offers and Declined offer). Topic 3: Overfitting vs Underfitting and 4 Methods to Prevent Overfitting

1. Overfitting vs Underfitting

Overfitting
Overfitting occurs when a model learns the training data too well. It captures not only the underlying patterns but also the random "noise" and outliers.

  • Performance: Extremely high accuracy on training data, but poor performance on new (test) data.
  • Analogy: A student who memorizes every specific practice question but fails the actual exam because they didn't understand the core concepts.

Underfitting
Underfitting occurs when a model is too simple to learn the underlying structure of the data.

  • Performance: Poor performance on both the training data and the test data.
  • Analogy: A student who only skims the table of contents and fails the exam because they don't know enough detail to answer any questions.

2. Four Methods to Prevent Overfitting

To ensure a model generalizes well and avoids overfitting, you can use the following techniques:

A. Cross-Validation
Instead of splitting data into just two parts (train and test), K-Fold Cross-Validation involves splitting the data into k subsets. The model is trained on k-1 subsets and validated on the remaining one. This process repeats k times, ensuring every data point is used for both training and validation.

  • Benefit: It provides a more robust estimate of how the model will perform on unseen data.

B. Regularization (L1 and L2)
Regularization adds a "penalty" term to the model's loss function based on the size of the coefficients (weights).

  • L1 (Lasso): Can shrink some coefficients to zero, effectively performing feature selection.
  • L2 (Ridge): Discourages large weights by squaring them in the penalty term.
  • Benefit: It prevents the model from becoming overly complex or relying too heavily on a single feature.

C. Pruning (Specific to Decision Trees)
Pruning is the process of removing branches from a decision tree that provide little power to classify instances.

  • Mechanism: You can prune "pre-emptively" (stop growing the tree early) or "post-pruning" (remove nodes after the tree is fully grown).
  • Benefit: It simplifies the model, making it less likely to memorize noise in the training set.

D. Increasing Training Data
One of the simplest ways to reduce overfitting is to provide the model with more examples.

  • Mechanism: With more data, the "noise" eventually averages out, and the model is forced to focus on the dominant, real patterns that persist across the larger dataset.
  • Benefit: It helps the model distinguish between a rare outlier and a general rule.

Feature Univariate Linear Regression Multivariate Linear Regression
