Evaluating a problem as a state space search

Definition: Evaluating a problem as a state space search involves defining the problem using five formal components. This creates a "state space" (a mathematical graph of all possible situations) that an AI can search through to find a sequence of actions leading to a goal.

The 5 Components of Problem Formulation:

  • Initial State: The starting situation of the agent.
  • Actions: The set of possible moves available to the agent in any given state.
  • Transition Model: A function that describes what each action does (Input: Current State + Action → Output: New State).
  • Goal Test: A condition applied to a single state to determine if it is the target destination.
  • Path Cost: A numeric function that assigns a cost to a sequence of actions. The AI aims to minimize this cost.

Example: 8-Queens Problem (Incremental Formulation)

Problem Description: The goal is to place 8 queens on a standard 8x8 chessboard so that no two queens attack each other (no two share the same row, column, or diagonal).

  • States: Any arrangement of 0 to 8 queens on the chessboard.
  • Initial State: An empty 8x8 board.
  • Actions: Add a single queen to any empty square. (Better abstraction: Add a queen to the leftmost empty column in a square that is not already under attack).
  • Transition Model: Returns the updated board with the new queen added to the specified square.
  • Goal Test: There are exactly 8 queens on the board, and none of them are attacking each other.
  • Path Cost: Zero. For this specific problem, we only care about the final correct arrangement, not the number of steps it took to get there.

Example: 8-Puzzle Problem

Problem Description: The 8-puzzle consists of a 3x3 board with eight numbered tiles and one blank space. A tile adjacent to the blank space can slide into the space. The objective is to reach a specific target configuration.

  • States: A state description specifies the location of each of the eight tiles and the blank space in one of the nine squares. (Total possible states: 9!/2 = 181,440).
  • Initial State: Any valid random starting arrangement of the numbered tiles and the blank space.
  • Actions: Instead of moving the tiles, we formulate the actions in terms of the blank space. The possible actions are: Up, Down, Left, Right.
  • Transition Model: Given a state and an action, this returns the resulting state. For example, if the action is Left, the model updates the board by swapping the position of the blank space with the tile immediately to its left.
  • Goal Test: This checks whether the current state matches the required target configuration (e.g., tiles ordered 1 through 8 with the blank in the bottom right corner).
  • Path Cost: Each single step (movement of the blank space) costs exactly 1. Therefore, the total path cost is the total number of moves taken to reach the goal state.
Topic 2: Tree Search vs Graph Search Strategies
  • Q41. List the differences between tree search algorithm and graph search algorithm. (3)
  • Q57. Discuss the difference between Tree Search and Graph search strategies for finding solutions. (8)

1. The Core Concept: Redundant Paths

When an AI agent searches through a state space, there is often more than one way to reach the same state. These are called redundant paths or cycles. How an algorithm handles these paths defines whether it is a Tree Search or a Graph Search.

2. Tree Search Strategy

  • How it works: It simulates exploring a tree by expanding nodes. It only keeps track of the Frontier.
  • The Problem: Tree Search has no "memory" of where it has been. It does not check if a newly generated node is a state it has already visited.
  • Consequence: In state spaces with cycles, a Tree Search can get trapped in an infinite loop.

3. Graph Search Strategy

  • How it works: It introduces a new data structure called the Explored Set (or "Closed List").
  • The Solution: Every time a node is expanded, it is added to the Explored Set. Before adding a newly generated node to the Frontier, the algorithm checks if it is already in the Frontier or the Explored Set. If it is, the node is discarded.
  • Consequence: It never explores the same state twice, preventing infinite loops.

4. Comparison Table

FeatureTree SearchGraph Search
Memory StructuresFrontier only.Frontier AND Explored Set.
Redundant PathsExplores them.Prunes them.
Cyclic EnvironmentsInfinite loops possible.Safe from infinite loops.
Space ComplexityLower memory.Higher memory.
CompletenessIncomplete in cyclic spaces.Always complete in finite spaces.

Heuristics function

  • Heuristic is a function used in Informed Search to find the most promising path.
  • It estimates how close an agent is from the goal, represented by h(n).
  • Admissibility Condition: h(n) ≤ h*(n). The heuristic should never overestimate the actual cost.

Example: Map Navigation

Heuristic (h(n)): The straight-line distance (as the crow flies) from the current city to the destination. This is always admissible because a straight line is the shortest possible distance.
